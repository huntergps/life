# Data Architecture

## Directory Layout

```
lib/data/
  local/drift/          # SQLite via Drift (offline storage)
    db/                  # Database definition, tables, converters, platform stubs
    adapters/            # Hand-written Supabase model dictionary
    repository/          # WildlifeRepository (offline-first entry point)
    drift.dart           # Barrel export
  remote/supabase/       # (reserved) Direct Supabase service calls
  sync/                  # Synchronisation services
    initial_sync_service.dart
    seed_data_service.dart
    realtime_service.dart
    image_preload_service.dart
    image_preload_provider.dart
  mappers/               # Row-to-model helpers
    data_helpers.dart
```

## Local Data Flow (Drift / SQLite)

`AppDatabase` (generated by Drift) defines 15 tables mirroring the Supabase schema. On native platforms it opens a file-backed SQLite database; on web it uses IndexedDB via Drift WASM.

`WildlifeRepository` extends `OfflineFirstWithSupabaseRepository` and is the single entry point for all data operations. Every query passes through Drift first, falling back to Supabase when the local cache is empty or stale.

## Remote Data Flow (Supabase)

Supabase acts as the remote source of truth. The repository's Supabase adapter layer (hand-written in `supabase_model_dictionary.dart`) maps between Drift row types and Supabase JSON.

Write operations are queued locally and pushed to Supabase when connectivity is available (offline queue).

## Offline Queue

`OfflineFirstWithSupabaseRepository` maintains a request queue in SQLite. When a mutation fails due to network issues, the request is stored and retried automatically when connectivity returns. The queue is FIFO per model.

## Sync Policies

| Policy | Behaviour |
|--------|-----------|
| `localOnly` | Read/write only the local Drift database. Used for user preferences and draft sightings. |
| `awaitRemote` | Always fetch from Supabase and update the local cache before returning. Used in admin screens. |
| `awaitRemoteWhenNoneExist` | Return local data if any rows exist; otherwise fetch from Supabase first. Default for species, islands, categories. |

## Seed Data (First Launch)

`SeedDataService` inserts a bundled dataset (categories, islands, visit sites, species, species-sites) directly into Drift so the app works fully offline on first launch without waiting for a Supabase sync. The seed runs only when the species table is empty.

Source data lives in `lib/core/data/seed_data.dart`.

## Initial Sync

`InitialSyncService` orchestrates the first full pull from Supabase after install. It syncs tables in dependency order (categories, islands, sites, species, images, species-sites, trails), then triggers image precaching. Progress is reported to the sync screen UI.

## Realtime Subscriptions

`RealtimeService` subscribes to Supabase Realtime channels for tables that change frequently (species, images, categories, islands, sites, species-sites, trails). On receiving an INSERT, UPDATE, or DELETE event it upserts or removes the corresponding row in the local Drift database, then invalidates relevant Riverpod providers so the UI updates immediately.

## Image Precaching

`ImagePreloadService` iterates all species with thumbnail URLs and downloads them into the HTTP cache (via `CachedNetworkImage`) so images display instantly when offline. `ImagePreloadProvider` exposes download progress to the UI.
